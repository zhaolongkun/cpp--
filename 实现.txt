====================================================
  反无人机追踪控制系统 — 实现记录
====================================================

一、项目目标
----------------------------------------------------
实现一套基于视觉的无人机自动追踪与电机控制系统：
- 摄像头实时采图
- YOLO 检测无人机目标
- 多目标跟踪（ByteTrack）
- 卡尔曼滤波平滑
- PID 控制两轴电机，使目标保持在画面中心

====================================================
二、整体数据流
====================================================

检测框 (YOLO)
  → ByteTrack MOT（多目标跟踪，Kalman内部平滑）
  → PNR-IMM-KF（卡尔曼滤波，输出 controlled_bbox）
  → 3级 EMA 低通滤波（串联，平滑抖动）
  → window=8/10 均值滤波
  → 中值滤波（window=3）
  → Temporal Compensator（GRU神经网络，补偿延迟）
  → fused_dx / fused_dy（最终控制误差）
  → PID 控制器（X轴、Y轴各一个）
  → 死区过滤（±10px 内发0）
  → clamp 限幅（±80）
  → NaN/Inf 安全检查
  → 电机 USB 命令（pyusb bridge）

====================================================
三、PID 控制实现
====================================================

1. 架构
   - PID 在 C++ 侧计算，电机接收速度命令（整数）
   - 两轴独立 PID：pid_x_、pid_y_
   - 电机通过 Python USB bridge 驱动

2. 参数（参考 Python 实现调整）
   pid_x: kp=0.07, ki=0.0, kd=0.0035
   pid_y: kp=0.06, ki=0.0, kd=0.004

3. 符号约定（与 Python 对齐）
   error_x = -dx_ctrl（目标在右，向右转）
   error_y = +dy_ctrl（Y轴反向）

4. 积分限幅（anti-windup）
   integral_ = clamp(integral_, -500, +500)
   防止 ki 非零时积分飞散

====================================================
四、安全防护机制
====================================================

1. armed: false       启动默认不驱动电机，必须手动开启
2. mode: mock         默认模拟模式，不发真实 USB 命令
3. cmd_limit: 80      PID 输出硬限幅 ±80
4. speed_limit: 80    执行器层再次限速
5. deadband_px: 10    小偏差直接发0，避免抖动
6. 积分限幅 500       防止积分飞散
7. NaN/Inf 检测       异常值清零并重置 PID
8. stop_when_lost     目标丢失立即停止并重置 PID
9. zoom_stable 条件   zoom 未稳定时不发命令

====================================================
五、延迟分析
====================================================

环节                        延迟估计
----------------------------------------------------
YOLO 检测                   10~30ms（硬延迟）
ByteTrack + KF              <1ms
3级 EMA + window均值        100~330ms（最大瓶颈）
跨线程 SharedState 传递     0~33ms
Temporal Compensator        补偿上述延迟（GRU预测）
----------------------------------------------------
总控制响应延迟              约 150~400ms

Temporal Compensator（seq_len=8 的 GRU）专门用于
预测目标未来位置，抵消 LPF 引入的平滑延迟。

====================================================
六、遇到的困难
====================================================

1. 架构理解偏差
   初期误以为电机接受位置误差、板上有自己的PID，
   后来确认：PID 在 C++ 侧，电机接收速度命令。
   影响：符号约定、控制量量纲需重新对齐。

2. 死区冲突
   曾尝试添加 lock_deadband（±5px）叠加在
   deadband_px（±10px）之上，逻辑冲突。
   解决：去掉 lock_deadband，统一使用 deadband_px=10。

3. std::clamp 编译错误
   pid.cpp 添加积分限幅时未包含 <algorithm>，
   MSVC 报 C2039/C3861 错误。
   解决：添加 #include <algorithm>。

4. 滤波延迟过大
   3级串联 EMA + window 均值引入约 300ms 延迟，
   导致控制响应偏慢。
   缓解：Temporal Compensator 神经网络预测补偿。

5. 多滤波层调试困难
   共5层滤波（ByteTrack KF、PNR-IMM-KF、3级EMA、
   中值滤波、GRU），中间状态多，定位问题复杂。
